# Vite Project and Component Library

## Goal

Set up the frontend build pipeline and shared Vue component library that produces the JS bundle embedded in HTML reports. Replace the stub `study/report/bundle.js` with a real Vite-built bundle containing Vue 3, Tailwind, ECharts, shadcn-vue, TanStack Table, and five domain-aware report components.

## Architecture

A Vite project lives in `study/ui/`. A `build.sh` script gathers `.vue` report files from study packages, generates an entry point that registers them all, runs the Vite build, and copies the output bundle to `study/report/bundle.js`. The committed bundle is what `go:embed` picks up -- `go build` does not require Node.js. Developers only run `build.sh` when they change frontend code.

The bundle is fully self-contained. No CDN dependencies. Reports work offline.

## Stack

- Vue 3
- Tailwind CSS (with Vite plugin, tree-shaken to used classes)
- shadcn-vue / Radix Vue (UI primitives for dropdowns, tooltips, etc.)
- ECharts (tree-shaken to used chart types)
- TanStack Table + TanStack Vue Virtual (headless table logic + virtual scrolling)

## Directory Structure

```

```

## Build Flow

`build.sh` performs these steps:

1. Create a temp build directory.
2. Find all `.vue` files under `study/` excluding `study/ui/src/components/`.
3. Copy them into the temp directory.
4. Generate `src/main.js` that:
   - Imports Vue and creates the app
   - Imports and registers each shared component
   - Imports and registers each report component (derived from gathered `.vue` files, component name from filename)
   - Reads `__REPORT_DATA__` and `__REPORT_COMPONENT__` globals
   - Mounts the app with dynamic component resolution
5. Run `vite build` with the temp directory as root.
6. Copy the output bundle to `study/report/bundle.js`.
7. Clean up the temp directory.

## Base Template Changes

`study/report/base.html` removes the three CDN script tags. The bundle contains everything. The template becomes:

```html
<!DOCTYPE html>
<html lang="en">
<head>
<meta charset="utf-8">
<meta name="viewport" content="width=device-width, initial-scale=1">
<title>{{.Title}}</title>
</head>
<body>
<div id="app"></div>
<script>
const __REPORT_DATA__ = {{.Data}};
const __REPORT_COMPONENT__ = "{{.Component}}";
</script>
<script>{{.Bundle}}</script>
</body>
</html>
```

## Design Language

The component library follows a design language derived from the portfolio holdings dashboard spec. Reports should feel like polished financial tools, not generic data dumps.

### Color Palette

Semantic colors used consistently across all components:

| Role | Hex | Usage |
|------|-----|-------|
| Positive / profitable | `#1D9E75` | Green returns, gains, favorable metrics |
| Negative / loss | `#E24B4A` | Red returns, losses, drawdowns |
| Warning / caution | `#BA7517` | Amber for values that warrant attention |
| Primary accent | `#378ADD` | Chart lines, links, interactive elements |
| Secondary accent | `#534AB7` | Purple for secondary data series |
| Tertiary accent | `#D85A30` | Orange for third data series |
| Foreground | `#1a1a1a` | Primary text |
| Muted | `#666` / `#999` | Labels, secondary text, axis labels |
| Surface | `#fafafa` | Card backgrounds, panel fills |
| Border | `#ddd` / `#eee` | Table borders, card borders, dividers |

These are configured in `tailwind.config.js` as named colors (`positive`, `negative`, `warning`, `accent`, etc.) so components use semantic class names like `text-positive` rather than raw hex values.

### Typography

- System font stack: `-apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif`
- Base size: 14px
- Section headings: 20px, font-weight 600, bottom border 2px solid foreground, 6px bottom padding
- Metric labels: 11px, uppercase, letter-spacing 1.5px, muted color
- Metric values: 20px, font-weight 600
- Table text: 13px
- Implementation notes / secondary info: 12px

### Spacing and Layout

- Page: max-width 900px, centered, 40px padding
- Sections: 48px top margin between major sections
- Panels: 1px solid border, 8px border-radius, 16px padding, surface background
- Metric strips: 5-column grid with 10px gap
- Two-column layouts via CSS grid with 12px gap for side-by-side panels
- Section labels: 11px uppercase with 1.5px letter-spacing, muted, 32px top margin, 6px bottom margin

### Chart Styling

- Chart series colors follow the palette: primary accent for strategy, secondary for benchmark, tertiary for additional series
- Grid lines: subtle (`rgba(0,0,0,0.06)`)
- Axis tick labels: muted color, 10px font
- Tooltips: solid background, clean typography, all series values shown at crosshair
- Legends: inline, 11px, muted, positioned above chart area

### Value Formatting Conventions

All components share the same formatting rules:
- Percentages: `12.34%` (2 decimal places), colored green/red by sign
- Currency: `$1,234.56` with comma thousands separator
- Large currency: compact notation (`$1.2M`, `$140k`) on chart axes
- Ratios: `1.234` (3 decimal places), colored green/red by sign
- Dates: `YYYY-MM-DD`
- Missing/NaN: muted "N/A" or empty cell depending on context

### Print Support

Reports should be print-friendly. Components respect `@media print` rules:
- Charts have `max-height: 250px` in print
- Page breaks between major sections via `.page-break` class
- Background colors preserved for heatmap cells

## Components

### FinancialTable

A financially-aware data table built on TanStack Table. Report authors pass columns with types and rows of data. The component handles all formatting, alignment, coloring, and interaction.

**Props:**
- `columns` -- array of `{key, header, type, align?}` where type is `percent`, `currency`, `ratio`, `date`, `string`, `number`
- `rows` -- array of objects keyed by column key
- `sortable` -- boolean, default true
- `pageSize` -- number, 0 for no pagination (default 0)
- `virtualScroll` -- boolean, enables vue-virtual for large datasets (default false)

**Behavior:**
- Right-aligns numeric/currency/percent columns automatically
- Formats values by column type: percentages with `%` and 2 decimal places, currencies with `$` and commas, ratios with 3 decimal places, dates as `YYYY-MM-DD`
- Colors positive values green, negative values red for numeric types
- Displays NaN/null as muted "N/A"
- Sortable column headers with sort direction indicator
- Optional comparison column type that shows +/- deltas with signed formatting
- Virtual scrolling via `@tanstack/vue-virtual` when enabled
- Responsive column sizing with minimum widths per type
- Styled with Tailwind: clean borders, alternating row shading, hover highlight

### TimeSeriesChart

A chart for financial time series built on ECharts.

**Props:**
- `series` -- array of `{name, times, values, style?}` where style is `solid` (default), `dashed`, or `area`
- `yAxisFormat` -- `currency` or `percent`
- `title` -- optional chart title
- `height` -- chart height in pixels (default 300)

**Behavior:**
- Date x-axis with intelligent tick intervals that adapt to the time range (daily for <1 month, monthly for <2 years, quarterly for longer)
- Dollar y-axis with compact labels ($1.2M, $140k) or percent y-axis (12.5%)
- Multiple series with auto-colored lines and legend
- Crosshair tooltip showing all series values at a given date
- Zoom and pan via ECharts dataZoom
- Dashed style for benchmark series
- Area fill for stacked composition views
- Responsive: fills container width, respects height prop

### HeatmapGrid

A value grid with diverging color scale. Used for monthly returns and any row-by-column numeric display.

**Props:**
- `rows` -- array of `{label, values}` where values is an array of numbers (NaN for empty cells)
- `columns` -- array of column header strings
- `format` -- `percent` or `number`
- `showTotals` -- boolean, compute and display row totals (default false)
- `title` -- optional title

**Behavior:**
- Diverging color scale: red (negative) through white (zero) to green (positive), with opacity proportional to magnitude
- Cell value formatting by type (percent shows `1.2%`, number shows raw value)
- NaN cells rendered as empty/blank
- Optional totals column computed as compound return (for percent) or sum (for number)
- Tooltip on hover showing exact value
- Responsive cell sizing with minimum width
- Row labels in a fixed left column

### PercentileFan

A chart for Monte Carlo simulation results showing percentile bands.

**Props:**
- `percentiles` -- object with keys `p5`, `p25`, `p50`, `p75`, `p95`, each an array of `{time, value}`
- `actual` -- optional array of `{time, value}` for the historical/actual series
- `yAxisFormat` -- `currency` or `percent` (default `currency`)
- `title` -- optional chart title
- `height` -- chart height in pixels (default 350)

**Behavior:**
- Five percentile series rendered as shaded bands: P5-P25 (outer, lightest), P25-P50, P50-P75, P75-P95 (outer, lightest)
- P50 median rendered as a solid line on top of the bands
- Gradient fills with decreasing opacity from median outward
- Optional actual/historical series rendered as a distinct colored line over the bands
- Date x-axis, dollar/percent y-axis with compact formatting
- Legend showing percentile levels and actual if present
- Tooltip showing all percentile values at a given date

### MetricCards

A horizontal strip of headline numbers.

**Props:**
- `metrics` -- array of `{label, value, format, comparison?, comparisonLabel?}`
- Format types: `currency`, `percent`, `ratio`, `number`, `string`

**Behavior:**
- Responsive grid: adapts columns to the count (3-6 cards per row)
- Value formatting by type: currency with `$` and commas, percent with `%`, ratio with decimal places
- Color coding: positive green, negative red for numeric types
- Optional comparison value shown below the main value as a smaller delta with +/- sign and arrow indicator
- NaN/null displayed as muted "N/A"
- Cards styled with Tailwind: subtle background, rounded corners, consistent spacing

## Vite Configuration

- `build.lib` mode to produce a single IIFE bundle (not ESM modules, since it's inlined into an HTML page via a script tag)
- Tailwind CSS plugin for utility class generation and tree-shaking
- ECharts tree-shaking: import only `LineChart`, `BarChart`, `HeatmapChart`, `ScatterChart` and required components (tooltip, legend, grid, dataZoom, visualMap)
- Output: single `.js` file, no code splitting, no source maps in production
- External: nothing -- fully self-contained bundle

## Testing

Components are tested by building the bundle and verifying it produces valid HTML when combined with the base template and sample data via the existing Go `report.Render` test. No Jest/Vitest unit tests for components in this phase -- the Go integration test is the verification path.

Manual verification: run `build.sh`, then use a Go test that calls `report.Render` with a test report, writes the output to a temp `.html` file, and opens it to confirm the components render correctly.
